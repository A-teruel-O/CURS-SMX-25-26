# T01: Control de versions amb git i GitHub

### Breu descripció

Des de que t’has incorporat a **EverPia** has vist com els treballadors usen una cosa anomenada **GitHub** i parlen de conceptes com repositoris, control de versions o *commits*. Paraules que et sonen molt estranyes i que t’han obert la curiositat.

Doncs bé, ara toca satisfer aquesta curiositat. Un dels consultors més veterans de l’empresa us ha convocat per fer una formació bàsica sobre el control de versions i l’ús de **git** i **GitHub**. En aquesta formació faràs les teves primeres passes amb l’ús d’una tecnologia fonamental dins el món de les tecnologies de la informació.

Veuràs la utilitat del control de versions, com funciona el protocol **git**, es parlarà dels tres estats (*working directory*, *staging* i *repository*) i veuràs com començar a usar aquesta tecnologia. També se’t parlarà del format **Markdown**, llenguatge de marcat que és la base de la documentació a GitHub i que caldrà que aprenguis per poder publicar les teves posteriors feines.

---

### Passar a l'acció

En acabar les explicacions, et tocarà passar a l’acció:

1. **Creació del compte:** Has de crear-te un compte a **GitHub**. Cal que usis un nom d’usuari que t’identifiqui clarament, per exemple `JoanAbella` o `JoanAB`, etc. Com a compte de correu has d’usar **obligatòriament** el correu de l’escola.
2. **Inicialització del repositori de perfil:** Un cop creat, crearem un primer repositori públic que haurà de tenir exactament el mateix nom que el teu nom d’usuari a GitHub.
3. **Documentació de presentació:** Dins d'aquest repositori crearem un arxiu que s’ha de dir `README.md` (cal respectar estrictament les majúscules). En aquest document escriureu sobre vosaltres, el vostre nom i què esteu estudiant. Penseu que això que escriureu serà com la vostra carta de presentació per a tothom que consulti el vostre GitHub.
